## Internationalization Rules

- All user-facing strings must use next-intl, never hardcode text in components
- Translation keys go in `messages/es.json` (primary) and `messages/en.json`
- Default locale is Spanish (es) — always add Spanish translations first
- Use `useTranslations('namespace')` in client components
- Use `getTranslations('namespace')` in server components
- Organize keys by feature: `pos.*`, `crm.*`, `inventory.*`, etc.
